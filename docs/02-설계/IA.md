## 문서 목적

본 문서는 성지제강 MES 프로토타입의 메뉴 구조, 화면 계층 구조, 역할별 접근 화면, URL 경로, 화면 간 데이터 흐름, 핵심 사용자 흐름을 정의합니다.  
각 메뉴와 화면은 `어떤 문제를 해결하기 위해 제공되는가`를 기준으로 구성합니다.

## 화면 계층 구조

```text
성지제강 MES
├── [대시보드]
│   ├── 생산 대시보드              /dashboard
│   └── 생산 현황판               /dashboard/board  (사이니지 전용 URL)
│
├── [생산관리]
│   ├── 생산실적 조회             /production/results
│   ├── 공정 진행 조회            /production/progress
│   └── 라벨 출력                 /production/label
│
├── [품질관리]
│   ├── 부적합 등록               /quality/register
│   ├── 부적합 분석               /quality/analysis
│   └── 성적서 발행               /quality/certificate
│
├── [재고/출하]
│   ├── 야적장 위치 조회          /inventory/yard
│   ├── 출하 현황 조회            /inventory/shipping
│   └── 원자재/로트 조회          /inventory/material
│
├── [추적관리]
│   └── 제작번호 추적 조회        /trace
│
├── [설비관리]
│   ├── 설비 상태 모니터링        /facility/status
│   ├── 설비 알림/안돈            /facility/alert
│   └── MTBF 조회                 /facility/mtbf      [비활성 — 2차 고도화]
│
└── [외주공정관리]
    └── 조립 외주 지시/상태 조회  /outsource
```

## 화면 목록

| 화면 ID | 화면명 | 상위 메뉴 | URL 경로 | 해결하려는 문제 | 제공 이유 |
| --- | --- | --- | --- | --- | --- |
| IA-01 | 생산 대시보드 | 대시보드 | /dashboard | 엑셀 중심 운영으로 생산 현황이 즉시 보이지 않음 | 공정별 생산실적과 진행 현황을 한 화면에서 보기 위해 |
| IA-02 | 생산 현황판 | 대시보드 | /dashboard/board | 사무실과 현장이 다른 기준으로 생산량을 봄 | 현장과 사무실이 동일한 생산량을 공유하기 위해 |
| IA-03 | 생산실적 조회 | 생산관리 | /production/results | 생산실적이 공정별로 흩어져 있음 | 일/주/월 기준 실적 집계와 비교 조회를 위해 |
| IA-04 | 공정 진행 조회 | 생산관리 | /production/progress | 제작번호 기준 진행 상황 파악이 어려움 | 제작번호별 현재 공정 상태를 바로 확인하기 위해 |
| IA-05 | 라벨 출력 | 생산관리 | /production/label | 공정별 라벨 출력 기준이 분절됨 | 작업지시 기준 라벨과 KS 라벨을 출력하기 위해 |
| IA-06 | 부적합 등록 | 품질관리 | /quality/register | 부적합 발생 시 수기 처리로 누락 가능성 존재 | 현장에서 즉시 부적합을 기록하기 위해 |
| IA-07 | 부적합 분석 | 품질관리 | /quality/analysis | 품질 실적과 추세를 보기 어려움 | 일/주/월 기준 품질 분석과 성적서 대응 기반을 만들기 위해 |
| IA-16 | 성적서 발행 | 품질관리 | /quality/certificate | 고객사별 성적서를 수기로 작성하여 오류와 지연 발생 | 고객사 양식에 생산/품질 데이터를 자동 삽입하여 PDF로 발행하기 위해 |
| IA-08 | 야적장 위치 조회 | 재고/출하 | /inventory/yard | 출하 시 제품 탐색 시간이 오래 걸림 | 현장명, 슬리퍼, 패킹번호 기준 위치를 빠르게 찾기 위해 |
| IA-09 | 출하 현황 조회 | 재고/출하 | /inventory/shipping | 적재와 출하 상태를 한 번에 보기 어려움 | 출하 준비와 진행 상황을 즉시 파악하기 위해 |
| IA-10 | 원자재/로트 조회 | 재고/출하 | /inventory/material | 선입선출과 로트 추적이 약함 | 입고, 사용, 재고 흐름을 로트 기준으로 관리하기 위해 |
| IA-11 | 제작번호 추적 조회 | 추적관리 | /trace | 불량 발생 시 역추적이 어려움 | 자재, 작업지시, 현장, 출하까지 연결 조회하기 위해 |
| IA-12 | 설비 상태 모니터링 | 설비관리 | /facility/status | 설비 상태를 사무실에서 즉시 알기 어려움 | TG기 접점 수집 정보와 주요 설비 가동/정지 상태를 확인하기 위해 |
| IA-13 | 설비 알림/안돈 | 설비관리 | /facility/alert | 설비 이상 발생 시 대응 공유가 늦음 | 안돈과 설비 이상 알림을 확인하기 위해 |
| IA-14 | MTBF 조회 | 설비관리 | /facility/mtbf | 설비 운영 지표가 체계적으로 관리되지 않음 | 설비별 가동/정지 이력을 지표로 확인하기 위해 — **⚠ 2차 고도화 대상** |
| IA-15 | 조립 외주 지시/상태 조회 | 외주공정관리 | /outsource | 외주 조립 공정 지시와 진행 관리가 끊김 | 외주업체 지시사항, 진행 상태, 후속 야적 흐름을 MES 안에서 연결하기 위해 |

## 내비게이션 구조

### 1. 운영 핵심 흐름

```
대시보드
  → 생산실적 조회 → 공정 진행 조회
  → 설비 상태 모니터링 → 설비 알림/안돈
  → 야적장 위치 조회 → 출하 현황 조회
```

### 2. 품질 흐름

```
공정 진행 조회 → 부적합 등록 → 부적합 분석
부적합 분석 → 제작번호 추적 조회
```

### 3. 추적/출하 흐름

```
제작번호 추적 조회 → 야적장 위치 조회 → 출하 현황 조회
원자재/로트 조회 → 제작번호 추적 조회
```

### 4. 외주 조립 흐름

```
공정 진행 조회 → 조립 외주 지시/상태 조회
조립 외주 지시/상태 조회 → 야적장 위치 조회
```

### 5. 설비 관리 흐름

```
설비 상태 모니터링 → 설비 알림/안돈
```

## 핵심 사용자 흐름 시나리오

### 시나리오 1: 생산관리자 — 일일 생산 현황 확인

```
[IA-01 생산 대시보드] 
  → KPI 이상값 발견 
  → [IA-04 공정 진행 조회] 제작번호 상세 확인 
  → 부적합 발생 건 확인 
  → [IA-07 부적합 분석] 추세 조회
```

### 시나리오 2: 현장 운영자 — 생산 후 라벨 출력 및 야적 등록

```
[IA-04 공정 진행 조회] 제작번호 확인 
  → [IA-05 라벨 출력] 라벨 발행 
  → 조립 완료 후 [IA-08 야적장 위치 조회] 위치 입력
```

### 시나리오 3: 설비 관리자 — 설비 정지 대응

```
[IA-12 설비 상태 모니터링] 이상 설비 확인 
  → [IA-13 설비 알림/안돈] 알림 상세 확인
```

### 시나리오 4: 출하 담당 — 고객 출하 처리

```
[IA-09 출하 현황 조회] 현장명/패킹번호 검색 
  → [IA-08 야적장 위치 조회] 위치 확인 
  → 출하 완료 상태 변경
```

### 시나리오 5: 품질 담당 — 불량 역추적

```
[IA-07 부적합 분석] 이상 건 발견 
  → [IA-11 제작번호 추적 조회] 해당 제작번호 이력 조회 
  → [IA-10 원자재/로트 조회] 투입 자재 확인
```

## 화면 상태 정의

| 상태 | 설명 | 적용 화면 |
| --- | --- | --- |
| 정상 | 실적, 상태, 위치, 이력이 정상 조회되는 상태 | 전체 화면 |
| 데이터 없음 | 조회 조건은 맞지만 데이터가 없는 상태 | IA-03, IA-09, IA-11 |
| 수집 지연 | 자동 수집 데이터가 아직 반영되지 않은 상태 | IA-01, IA-12 |
| 수동 보정 필요 | 자동 수집 실패로 수동 입력 또는 보정이 필요한 상태 | IA-03, IA-04 |
| 알림 발생 | 설비 정지, 안돈, 품질 이상 알림이 있는 상태 | IA-13 |
| 권한 제한 | 역할상 수정 또는 특정 기능 접근이 제한된 상태 | IA-05, IA-06, IA-15 |

## 특수 접근 화면

| 화면 | URL | 용도 | 인증 방식 |
| --- | --- | --- | --- |
| 생산 현황판 | /dashboard/board | 공장 사이니지 화면 (TV/모니터 전용) | 전용 토큰 or IP 기반 접근 허용 |
| 모바일 최적화 화면 | /m/* | 분진 환경 내 태블릿/패드 접근 | 동일 JWT 인증 |

## 미해결 이슈

- 야적장 메뉴 명칭을 `재고/출하`로 둘지 `야적장/출하`로 직접화할지 결정 필요
- 외주공정관리 화면을 독립 메뉴로 둘지 생산관리 하위로 둘지 결정 필요
- TG 관련 조회를 생산관리 안에 둘지 설비관리와 연결 탭으로 둘지 추후 검토 필요
- 품질팀 알림과 성적서 대응 메뉴를 품질관리 안에서 별도 분리할지 추후 검토 필요
- 모바일/태블릿 최적화 화면 범위 확정 필요 (분진 환경 단말기 고려)
