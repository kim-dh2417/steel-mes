## 문서 목적

본 문서는 성지제강 MES 프로토타입의 대표 화면을 정의합니다.  
각 화면은 `어떤 문제를 해결하기 위해 제공되는가`, `무엇을 보여야 하는가`, `사용자는 어떤 행동을 하는가`, `필드 상세 명세`, `API 연계`를 기준으로 정리합니다.

## SCR-01 생산 대시보드

### 화면 개요

- **목적**: 생산실적, 공정 진행, 품질, 설비 상태를 한 화면에서 요약 조회
- **접근 역할**: 전 역할 (ADMIN, PROD_MANAGER, OPERATOR, QUALITY, FACILITY, SHIPPING)
- **해결 문제**: 부서별로 따로 보던 현황을 하나의 기준 화면으로 통합
- **데이터 갱신 주기**: 60초 자동 갱신

### 레이아웃 구성

```
┌──────────────────────────────────────────────────────────────┐
│ [상단] KPI 카드 4개                                         │
│ - 금일 총 생산량 / 목표 달성률 / 부적합 건수 / 설비 정지(30분+) │
├──────────────────────────────────┬───────────────────────────┤
│ [좌측 메인] 시간대별 생산량 추이 │ [우측] 주요 설비 상태     │
│ - 금일 기준 공정별 누계          │ - TG 핵심 설비 상태        │
│ - 공정별 라인 추이 확인          │ - 이상 설비 즉시 표시      │
├──────────────────────┬──────────────────────┬──────────────┤
│ [좌하] 공정별 생산량 비교 │ [중하] 공정별 목표 달성률 │ [우하] 최근 알림 │
│ - 오늘/어제 비교         │ - 공정별 실적/목표값      │ - 최근 알림 4건  │
│ - 공정별 생산 수량 비교  │ - 프로그레스 바           │ - 알림 상세 이동 │
└──────────────────────────────────────────────────────────────┘
```

### 컴포넌트 명세

| 컴포넌트명 | 유형 | 표시 데이터 | 클릭 동작 |
| --- | --- | --- | --- |
| KPI 카드 — 금일 총 생산량 | 숫자 카드 | 공정별 합계 수량 | /production/results 이동 |
| KPI 카드 — 목표 달성률 | 숫자 카드 | 금일 누계 / 목표값 | /production/results 이동 |
| KPI 카드 — 부적합 건수 | 숫자 카드 | 금일 등록 건수 | /quality/analysis 이동 |
| KPI 카드 — 설비 정지(30분+) | 숫자 카드 | 30분 이상 정지 설비 건수 | /facility/alert 이동 |
| 설비 상태 카드 | 상태 배지 그리드 | 설비명 + 가동/정지 상태 | /facility/status?facilityId= |
| 시간대별 생산량 추이 차트 | 라인 차트 | 금일 기준 공정별 누계 추이 | - |
| 공정별 목표 달성률 | 프로그레스 바 | 공정별 실적 / 목표값 | - |
| 공정별 생산량 비교 차트 | 막대 차트 | 오늘/어제 공정별 생산 수량 비교 | - |
| 최근 알림 | 리스트 | 최근 알림 4건 | 해당 알림 상세 이동 |

### API 연계

| API | Method | 설명 | 갱신 주기 |
| --- | --- | --- | --- |
| /api/dashboard/summary | GET | KPI 카드 집계 데이터 | 60초 polling |
| /api/facility/status | GET | 설비 가동/정지 상태 | 30초 polling |
| /api/alerts/recent | GET | 최근 알림 5건 | 60초 polling |

### 상태 처리

| 상태 | 표시 방법 |
| --- | --- |
| 수집 지연 | KPI 카드 우상단 `수집 지연` 배지 |
| 설비 정지 | 해당 설비 카드 빨간색 강조 + 정지 시간 표시 |
| 알림 없음 | 중요 알림 피드 영역 `금일 알림 없음` 안내 |

### 개발 유의사항

- 대시보드는 상세 입력 화면이 아니라 요약/이동 허브 역할을 유지합니다.
- 생산 현황판(SCR-01-B)과 동일한 데이터를 사용하지만, 대시보드는 관리자용 정보 밀도를 높이고, 현황판은 현장 사이니지용으로 단순화합니다.
- 분진 환경 태블릿에서도 터치로 카드를 탭할 수 있어야 합니다 (최소 터치 영역 48px).

## SCR-01-B 생산 현황판 (사이니지 전용)

### 화면 개요

- **목적**: TV/대형 모니터에 상시 표시되는 현장 현황 화면
- **접근 방식**: 전용 URL `/dashboard/board` — 로그인 없이 IP 또는 토큰 기반 접근
- **데이터 갱신 주기**: 60초 자동 갱신

### 표시 항목

| 항목 | 표시 형식 | 갱신 |
| --- | --- | --- |
| 공정별 금일 생산량 | 대형 숫자 | 60초 |
| TG 설비 7대 상태 | 신호등 색상 카드 | 30초 |
| 조립 5개 라인 생산량 | 수평 막대 | 60초 |
| 현재 시각 | 우상단 고정 | 1초 |

## SCR-02 생산실적 조회

### 화면 개요

- **목적**: 공정별 생산실적을 기간/설비 기준으로 상세 조회하고 일마감 실적을 확인
- **접근 역할**: OPERATOR, PROD_MANAGER, QUALITY (조회), FACILITY (조회)
- **해결 문제**: 실적이 공정별로 흩어져 있어 기간별 비교·보고가 어려움

### 레이아웃 구성

```
┌──────────────────────────────────────────────────────┐
│  [상단] 기간 선택 | 공정 필터 | 설비 필터 | 집계 단위│
├──────────────────────────────────────────────────────┤
│  [중단] KPI 요약 카드                                │
│  총 생산량 | 공정별 생산량 | 전일 대비 | 목표 달성률 │
├──────────────────────┬───────────────────────────────┤
│  [좌하] 시간대별     │  [우하] 공정별 비교 차트     │
│  생산량 차트         │  신선/TG/포밍/조립 막대 비교 │
├──────────────────────┴───────────────────────────────┤
│  [하단] 실적 상세 테이블 (날짜 × 공정 × 설비)       │
└──────────────────────────────────────────────────────┘
```

### 컴포넌트 명세

| 컴포넌트명 | 유형 | 표시 데이터 |
| --- | --- | --- |
| KPI 카드 — 총 생산량 | 숫자 카드 | 기간 내 합계 (m²) |
| KPI 카드 — 목표 달성률 | 숫자 카드 | 목표 대비 % (PRD D-02) |
| 시간대별 생산량 차트 | 라인/막대 차트 | 시간대별 누계 |
| 공정별 비교 차트 | 가로 막대 차트 | 공정별 생산량 비교 |
| 실적 상세 테이블 | 데이터 테이블 | 날짜 / 공정 / 설비 / 생산량 / 보정 여부 |

### 필드 명세 — 조회 조건

| 필드명 | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| 기간 | 날짜 범위 | 금일 | 일/주/월 단위 탭 전환 |
| 공정 | 다중 선택 | 전체 | 신선/TG/포밍/조립 |
| 설비 | 다중 선택 | 전체 | 공정 선택 연동 |
| 집계 단위 | 선택 | 시간별 | 시간별 / 일별 / 주별 |

### 필드 명세 — 실적 상세 테이블

| 컬럼명 | 데이터 | 설명 |
| --- | --- | --- |
| 일시 | datetime | 집계 단위 기준 시점 |
| 공정 | processName | 신선/TG/포밍/조립 |
| 설비 | facilityName | 설비명 |
| 생산량 | quantity | m² 또는 개 |
| 수집 방식 | dataSource | 자동(OPC/접점) / 수동 보정 |
| 보정 여부 | isCorrected | Y/N — 수동 보정 건 강조 |

### API 연계

| API | Method | 설명 |
| --- | --- | --- |
| /api/production/results | GET | 기간/공정/설비 기준 집계 조회 |
| /api/production/results/export | GET | 엑셀 다운로드 |

### 상태 처리

| 상태 | 표시 방법 |
| --- | --- |
| 수동 보정 건 | 해당 행 주황색 배경 + `보정` 배지 |
| 데이터 없음 | 빈 상태 메시지 + 조회 조건 재확인 안내 |

### 개발 유의사항

- 목표 수량은 작업지시 기준으로 공정별 목표를 설정해야 하며, 목표 마스터 관리 화면이 추후 필요합니다.
- 엑셀 다운로드는 일마감 보고 자동화 요건(PRD A-05)을 지원합니다.

## SCR-03 공정 진행 조회

### 화면 개요

- **목적**: 제작번호 기준으로 현재 어느 공정까지 진행됐는지 조회
- **접근 역할**: OPERATOR, PROD_MANAGER, QUALITY, FACILITY (조회)
- **해결 문제**: 생산품 상태와 공정 진행 정보를 즉시 알기 어려움

### 레이아웃 구성

```
┌──────────────────────────────────────────────────────┐
│  [상단] 검색: 제작번호 | 현장명 | 공정 상태 필터    │
├──────────────────────────────────────────────────────┤
│  [본문] 제작번호 목록 테이블 (페이지 20건)           │
│  제작번호 | 현장명 | 신선 | TG | 포밍 | 조립 | 상태 │
├──────────────────────────────────────────────────────┤
│  [우측 패널] 선택 건 상세 (Slide-in)                │
│  - 공정별 상세 실적 / 마지막 생산품 정보            │
│  - 부적합 등록 버튼 / 추적 조회 이동 버튼           │
└──────────────────────────────────────────────────────┘
```

### 필드 명세 — 목록 테이블

| 컬럼명 | 데이터 | 정렬 | 필터 |
| --- | --- | --- | --- |
| 제작번호 | workId | O | 텍스트 검색 |
| 현장명 | siteName | O | 텍스트 검색 |
| 신선 상태 | process1Status | X | 상태 선택 |
| TG 상태 | process2Status | X | 상태 선택 |
| 포밍 상태 | process3Status | X | 상태 선택 |
| 조립 상태 | process4Status | X | 상태 선택 |
| 전체 상태 | overallStatus | O | 상태 선택 |
| 최종 실적 시간 | lastUpdatedAt | O | - |

### 공정 상태 배지 색상

| 상태 코드 | 표시 | 색상 |
| --- | --- | --- |
| PS-01 대기 | 대기 | 회색 |
| PS-02 진행중 | 진행중 | 파란색 |
| PS-03 완료 | 완료 | 초록색 |
| PS-04 보정 필요 | 보정 필요 | 주황색 |

### API 연계

| API | Method | 설명 |
| --- | --- | --- |
| /api/production/progress | GET | 공정 진행 목록 (페이지네이션) |
| /api/production/progress/{workId} | GET | 제작번호 상세 |

### 상태 처리

- 데이터 없음: 검색 결과 없음 안내 + 검색 조건 초기화 버튼
- 보정 필요 건: 해당 행 주황색 배경으로 강조

### 개발 유의사항

- 진행 상태는 `대기 / 진행중 / 완료 / 보정 필요` 4가지만 사용합니다.
- 상세 패널에서 부적합 등록 이동 시 제작번호와 공정 정보를 URL 파라미터로 전달합니다.

## SCR-04 부적합 등록/분석

### 화면 개요

- **목적**: 부적합을 등록하고 품질 실적을 분석
- **접근 역할**: OPERATOR (등록), QUALITY/PROD_MANAGER (분석 포함)
- **해결 문제**: 수기 부적합 관리와 월 단위 수동 보고

### 레이아웃 구성

```
┌──────────────────────────────────────────────────────┐
│  [탭] 부적합 등록 | 부적합 분석                     │
├──────────────────────────────────────────────────────┤
│  [탭: 등록]                                         │
│  공정 선택 | 제작번호 | 부적합 유형 | 수량/중량     │
│  메모 | [저장] 버튼                                 │
├──────────────────────────────────────────────────────┤
│  [탭: 분석]                                         │
│  기간 선택 | 공정 필터                              │
│  부적합 건수 차트 | 유형별 분포 | 부적합률           │
│  상세 목록 테이블                                   │
└──────────────────────────────────────────────────────┘
```

### 필드 명세 — 부적합 등록 폼

| 필드명 | 타입 | 필수 | 유효성 | 비고 |
| --- | --- | --- | --- | --- |
| 공정 | 선택 (드롭다운) | Y | TG/포밍/조립 중 선택 | 신선 부적합은 별도 확인 필요 |
| 제작번호 | 텍스트 | Y | 존재 여부 실시간 검증 | 작업지시와 연동 |
| 부적합 유형 | 선택 (드롭다운) | Y | NC-01~NC-05 | 코드 마스터 관리 |

## SCR-10 조립 외주 지시/상태 조회

### 화면 개요

- **목적**: 외주 조립 공정 지시사항과 상태를 내부에서 관리
- **접근 역할**: PROD_MANAGER (등록+조회), OPERATOR (상태 확인)
- **해결 문제**: 외주 공정 지시와 진행 정보 단절

### 레이아웃 구성

```
┌──────────────────────────────────────────────────────┐
│  [상단] 필터: 상태 | 현장명 | 기간 / [+ 지시 등록]  │
├──────────────────────┬───────────────────────────────┤
│  [좌] 작업 목록      │  [우] 선택 건 상세            │
│  - 상태 배지 강조    │  - 지시 내용                  │
│  - 지연 건 상단 정렬 │  - 완료 예정일                │
│                      │  - 상태 변경 버튼             │
│                      │  - [야적장 위치 등록] 이동    │
└──────────────────────┴───────────────────────────────┘
```

### 필드 명세 — 지시 등록 폼

| 필드명 | 타입 | 필수 | 유효성 |
| --- | --- | --- | --- |
| 제작번호 | 텍스트 | Y | 작업지시 존재 검증 |
| 현장명 | 자동 연동 | Y | 제작번호 기준 자동 입력 |
| 지시 내용 | 텍스트 (멀티라인) | Y | 최대 1000자 |
| 완료 예정일 | 날짜 | Y | 오늘 이후 날짜만 허용 |

### 개발 유의사항

- 1차 범위는 내부 관리 화면 수준이며, 외주업체 포털은 미구현입니다.
- 완료 처리 후 야적장 위치 등록 화면(SCR-04)으로 이동 버튼을 제공합니다.
- 외주업체 직접 접근 여부는 2차 고도화 시 결정합니다.
