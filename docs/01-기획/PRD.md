# 목표

1. 데크 3공장의 **수동 엑셀 운영을 MES로 전환**하여 생산 현황을 실시간으로 파악한다.
2. 공정별 생산실적을 **자동 집계**하여 일마감/주간/월간 보고 자동화를 실현한다.
3. 불량품 발생 시 **원자재→공정→출하까지 역추적**이 가능한 이력 체계를 구축한다.
4. 야적장 제품 위치를 **현장명+슬리퍼 번호 기준으로 실시간 조회**하여 출하 소요 시간을 단축한다.
5. 설비 정지 및 품질 이상을 **즉시 알림**하여 대응 지연을 줄인다.
6. 3공장 MES 성공 후 **타 사업부 확산 및 ERP 연동** 기반을 마련한다.

# 제품 개요

성지제강 데크 3공장 MES는 **신선 → TG → 성형 → 조립 → 출하**의 전 공정에 걸쳐 생산실적, 설비 상태, 품질, 재고를 통합 관리하는 웹 기반 제조 실행 시스템이다.

| 항목 | 내용 |
|---|---|
| 대상 공장 | 성지제강 데크 3공장 (충북 음성) |
| 주력 제품 | 데크 플레이트 (주문 제작, 100여 종) |
| 생산 방식 | 수주 후 1~2주치 분할 납품, 슬리퍼 단위 관리 |
| 공정 수 | 4개 공정 (신선 / TG / 성형 / 조립) |
| 조립 공정 | 외주업체 위탁 운영 (5개 라인, 주야 각 5라인) |
| 현재 IT 현황 | 전산 시스템 없음, 전부 엑셀/이메일 운영 |
| 인프라 현황 | 서버실 없음, 사무동 이전 후 구성 예정 |
| 환경 제약 | 철가루 분진 극심 → 현장 내 PC 설치 불가 |
| 확장 목표 | ERP 연동 → 전사 확산 → IPO |

# 제품 방향

- **자동화 우선**: 수동 입력을 최소화하고 OPC·접점 신호·센서 기반 자동 집계를 기본으로 한다.
- **현실적 구조**: 완전 자동화보다 운영 가능한 자동집계 구조를 우선한다. TG기는 접점 신호 기반 간접 집계를 적용한다.
- **현장 환경 대응**: 분진 환경을 고려해 현장 내 PC 설치 대신 원격 접속 + 분진 내구성 단말기 구조로 설계한다.
- **확장 가능한 데이터 구조**: 이후 ERP 연동을 고려해 원자재→생산→출하의 전 과정을 하나의 추적 키로 연결한다.
- **외주 공정 통합**: 조립 외주업체를 MES 안으로 통합하여 작업 지시 전달 및 진행 모니터링을 시스템화한다.

# 핵심 시나리오

## 1. TG기 생산실적 자동 수집

TG기에 부착된 접점 신호 장치가 1회 타격(200mm 기준)마다 카운트 신호를 발생시킨다.
MES가 이 신호를 수집하여 생산량을 역산하고 제작번호에 자동 매핑한다.
생산 관리자는 대시보드에서 TG기별 실시간 생산량을 확인한다.

## 2. 불량 발생 후 역추적

성형 공정에서 부적합이 발생하면 OP가 부적합 등록 화면에서 제작번호와 불량 내용을 입력한다.
추적관리 화면에서 해당 제작번호를 조회하면, 투입된 원자재 로트번호, 작업지시 내용, 수주 현장명까지 역으로 확인할 수 있다.
품질팀에 알림이 자동 발송된다.

## 3. 야적장 출하 처리

조립 공정 완료 후 포장 단계에서 출하팀이 위치번호(현장명 + 슬리퍼 번호)를 입력한다.
고객 출하 요청 시 출하팀은 야적장 조회 화면에서 해당 현장의 슬리퍼 위치를 즉시 확인하고 지게차에 안내한다.

## 4. 설비 이상 알림

설비 이상이나 안돈 신호가 발생하면 시스템이 생산 관리자와 공무팀에 즉시 알림을 발송한다.
담당자는 설비 상태 화면에서 이상 설비와 발생 시각을 확인하고 대응을 시작한다.

## 5. 외주 조립 작업 지시

생산 관리자가 MES에서 조립 작업 지시를 생성하면, 외주 운영자의 화면에 작업 내용(제작번호, 수량, 납기)이 표시된다.
외주 운영자는 공정 진행 상황을 입력하고, 사무실에서 실시간으로 진행률을 확인한다.

## 6. KS 라벨 발행

신선 공정 완제품 생산 시 OP가 라벨 출력 화면에서 해당 제작번호를 선택하면,
KS 인증 필수 항목(규격, 로트번호, 생산일자 등)이 자동으로 삽입된 라벨이 출력된다.

# MVP 범위

## 포함 범위

- 생산실적 자동 집계 (OPC + TG 접점 신호 기반)
- 제작번호별 공정 진행 조회
- 부적합 등록 및 불량실적 관리
- 공정별 라벨 출력 (KS 라벨 포함)
- 생산 현황판 (사무실 + 현장)
- 야적장 재고 위치 관리 (현장명 + 슬리퍼 번호 기준)
- 적재/출하 현황 조회
- 추적관리 (원자재→출하 역추적)
- 원자재 로트관리 (선입선출)
- 안돈 및 설비 이상 알림
- 외주 공정 작업 지시 및 진행 모니터링
- 삼성/현대 등 주요 고객사 성적서 자동 발행

## 제외 범위

- MTBF 레포팅 → 2차 고도화
- 설비 30분 정지 자동 알림 및 정지유형 입력 → 2차 고도화
- 설비별 공정전환횟수 집계 → 2차 고도화
- Location Map 시각화 (야적장 구획 지도) → 2차 고도화
- 오프라인 야적장 세부 구획 작업 → 2차 고도화
- 오프라인 부적합 측정 저울 데이터 연동 → 가능 여부 검토 후 결정
- ERP 연동 → 2차 이후
- 타 사업부 확산 → 3공장 성공 후

# MVP 경계 메모

- TG기 접점 신호 수집은 1차 MVP의 핵심 전제 조건이다. OPC 연결은 이탈리아 제조사 비공개 정책으로 불가하므로, 접점 방식으로 구현 범위를 확정한다.
- 야적장 Location Map(지도 기반 시각화)은 1차에서 제외하되, 위치 데이터 구조는 2차 고도화를 고려하여 설계한다.
- 분진 환경으로 인해 현장 내 PC 설치가 불가하므로, 현장 단말기는 분진 내구성 제품으로 별도 구성한다.
- 서버 인프라는 사무동 이전 일정과 연계하여 결정한다. 이전 전에는 클라우드 임시 환경을 고려할 수 있다.

|---|---|---|
| A-01 | 공정별(신선/TG/성형/조립) 생산실적을 자동 집계한다 | OPC 또는 접점 신호로 실시간 카운트, 5분 이내 MES 반영 |
| A-02 | TG기는 접점 신호 기반 카운트로 생산량을 역산한다 | 1회 타격 간격 기준 카운트, 생산량 = 카운트 × 단위 길이 |
| A-03 | 제작번호별 공정 진행 현황을 실시간 조회한다 | 신선/TG/성형/조립 진행 상태를 한 화면에서 확인 가능 |
| A-04 | 자동 집계 실패 시 수동 보정 입력을 제공한다 | OP가 수동으로 실적 보정 가능, 보정 이력 기록 |
| A-05 | 일마감 기준 공정별 생산실적을 자동 산출한다 | 마감 시점 기준 누계 자동 집계 및 저장 |
| A-06 | 마지막 생산품 확인 기능을 제공한다 | 공정별 가장 최근 생산 제작번호 및 수량 확인 가능 |

## B. 부적합 관리

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| B-01 | TG/성형 공정에서 부적합 발생 시 즉시 등록한다 | 제작번호, 공정, 부적합 유형, 수량/중량 입력 가능 |
| B-02 | 중량 기반 부적합률을 자동 계산한다 | 부적합 중량 / 생산 총 중량 × 100 자동 산출 |
| B-03 | 부적합 이력을 일/주/월 단위로 분석한다 | 기간별 부적합 현황 차트 및 테이블 제공 |
| B-04 | 부적합 발생 시 품질팀에 알림을 발송한다 | 등록 즉시 QUALITY 역할에 알림 전송 |
| B-05 | 주요 고객사(삼성/현대 등) 성적서를 자동 발행한다 | 고객사별 양식에 데이터 자동 삽입, PDF 출력 |

## C. 라벨 출력

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| C-01 | 작업지시서 기준 공정별/설비별 라벨을 출력한다 | 제작번호, 공정, 규격, 로트번호 포함 |
| C-02 | 신선 공정 완제품에 KS 라벨을 자동 발행한다 | KS 필수 항목(규격, 생산일자, 로트번호 등) 자동 삽입 |
| C-03 | 공정별 지정 라벨프린터를 통해 출력한다 | TG 3대, 성형 2대, 조립 2대, 신선 1대, 오피스 패드 8대 |

## D. 생산 현황판

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| D-01 | 사무실과 현장에서 TG 및 조립 생산량을 실시간 확인한다 | 최대 1분 이내 갱신, 사무실 + 현장 현황판 동시 표시 |
| D-02 | 공정별 목표 대비 달성률을 표시한다 | 목표 수량 대비 현재 생산 수량 비율 표시 |

## E. 야적장 재고 및 출하 관리

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| E-01 | 패킹 완료 후 현장명 + 슬리퍼 번호 기준 위치를 입력한다 | 조립 마지막 단계에서 위치번호 입력 가능 |
| E-02 | 패킹번호(P/N)별 야적장 위치를 실시간 조회한다 | 현장명/슬리퍼 번호 검색으로 위치 즉시 확인 |
| E-03 | 적재 재고 및 출하 현황을 실시간 조회한다 | 입고/출고 상태별 재고 목록 제공 |

## F. 추적관리

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| F-01 | 제작번호 기준으로 원자재→철선→TG→조립→출하 이력을 연결한다 | 하나의 추적 키로 전 공정 이력 조회 가능 |
| F-02 | 불량 발생 시 투입 원자재, 작업지시, 수주 현장까지 역추적한다 | 불량 제작번호로 원자재 로트번호, 현장명 확인 가능 |
| F-03 | 프로젝트(현장) 단위로 추적 결과를 확인한다 | 현장명 기준 관련 제작번호 전체 조회 가능 |

## G. 원자재 수불 및 로트관리

| ID | 요구사항 | 수용 기준 |
|---|---|---|
| G-01 | 원자재 입고 시 라벨을 부착하고 로트번호를 부여한다 | 입고 등록 시 로트번호 자동 생성, 라벨 출력 연동 |
| G-02 | 선입선출(FIFO) 기반으로 원자재를 불출한다 | 동일 품목 중 먼저 입고된 로트 우선 출고 알림 |
| G-03 | 원자재 입고/출고/잔량 이력을 조회한다 | 로트번호 기준 수불 이력 전체 조회 가능 |

## H. 설비 알림 및 관리

|---|---|---|

| H-02 | 설비별 설비 상태와 알림 이력을 조회한다 | 설비명, 상태, 발생 시각, 최근 알림 이력 제공 |

# 이벤트 및 알림 모델

## 핵심 이벤트 유형

| 이벤트 유형 | 발생 조건 | 수신 역할 | 심각도 |
|---|---|---|---|
| 설비 고장 | 안돈 신호 발생 | 생산 관리자, 공무팀 | Critical |
| 부적합 발생 | 부적합 등록 완료 | 품질팀 | Medium |
| 외주 지연 | 납기 대비 진행률 미달 | 생산 관리자 | Medium |
| 원자재 부족 | 원자재 재고 기준량 이하 | 생산 관리자 | Low |

## 이벤트 공통 속성

| 속성 | 설명 |
|---|---|
| event_id | 이벤트 고유 ID |
| event_type | 이벤트 유형 코드 |
| severity | 심각도 (Critical / High / Medium / Low) |
| source | 발생 설비 또는 공정 |
| created_at | 발생 시각 |
| status | 미확인 / 확인 / 조치 완료 |
| resolved_at | 조치 완료 시각 |

## 심각도 정의

| 심각도 | 정의 | 대응 시간 |
|---|---|---|
| Critical | 생산 전면 중단, 즉시 대응 필요 | 즉시 (10분 이내) |
| High | 생산 지연 발생, 빠른 대응 필요 | 30분 이내 |
| Medium | 품질/진행 이상, 당일 대응 | 당일 |
| Low | 정보성 알림, 모니터링 수준 | 다음 업무일 |

## 이벤트 상태 정의

| 상태 | 설명 |
|---|---|
| 미확인 | 이벤트 발생, 담당자 미확인 |
| 확인 | 담당자가 이벤트 확인 |
| 조치 완료 | 담당자가 조치 완료 처리 |

# 기술 전제

| 항목 | 내용 |
|---|---|
| TG기 데이터 수집 | OPC 불가 (이탈리아 제조사 비공개) → 접점 신호 기반 카운트 |
| 신선/성형/조립 수집 | PLC 접근 가능 → OPC UA 기반 자동 집계 |
| 현장 단말기 | 분진 환경 → IP 등급 분진 내구성 터치 단말기 |
| 서버 환경 | 사무동 이전 후 서버실 구성, 이전 전 클라우드 임시 운영 고려 |
| 레거시 시스템 | 없음 → 완전 신규 구축 |
| 기존 데이터 | 엑셀 데이터 마이그레이션 범위 별도 결정 필요 |
| 확장 고려 | 이후 ERP 연동을 고려한 데이터 구조 설계 |
