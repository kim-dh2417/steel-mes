## 문서 목적

본 문서는 성지제강 데크 3공장 MES 프로토타입의 핵심 업무 프로세스를 정의합니다.  
각 프로세스는 `현재 문제(As-Is)`, `목표 상태(To-Be)`, `MES 처리 흐름`, `담당 역할`, `예외 및 확인 포인트`를 기준으로 정리합니다.

## 프로세스 간 연계 구조

```
원자재 입고 (PROC-05 시작)
    ↓ [로트 생성 → 입고 라벨]
신선 공정 → TG 공정 → 포밍 공정
    ↓ [PROC-01: 생산실적 자동 집계]
    ↓ [PROC-02: 부적합 발생 시 등록]
    ↓ [PROC-03: 공정별 라벨 출력]
조립 공정 (외주)
    ↓ [PROC-07: 외주 지시 / 진행 확인]
야적장 적재
    ↓ [PROC-04: 위치 등록 / 출하 조회]
출하
    ↓ [PROC-05: 추적 키 마감]
```

연계 데이터 키: `제작번호 (작업지시 ID)` → 공정 전체를 관통하는 단일 추적 키

## PROC-01 생산실적 집계 및 공정 진행 관리

### As-Is / To-Be

| 구분 | As-Is | To-Be |
| --- | --- | --- |
| 실적 수집 | 수기 입력 및 엑셀 정리 | OPC/접점 신호 기반 자동 집계 |
| 집계 주기 | 일마감 수동 | 실시간 또는 공정 완료 시점 즉시 갱신 |
| 진행 조회 | 불가 (엑셀 별도 확인) | 제작번호 기준 공정 진행 상태 실시간 조회 |
| 현장-사무실 공유 | 전화/이메일 | 생산 현황판 공유 |

### 담당 역할

| 단계 | 역할 |
| --- | --- |
| 생산 이벤트 발생 | 설비(자동) 또는 OPERATOR |
| 실적 확인 및 보정 | OPERATOR |
| 현황 조회 | PROD_MANAGER, VIEWER |
| 일마감 확정 | PROD_MANAGER |

### MES 처리 흐름

| 단계 | 행위자 | 동작 | 생성/변경 데이터 |
| --- | --- | --- | --- |
| 1 | 설비(자동) | 생산 이벤트 발생 (접점 신호 or OPC) | 생산 이벤트 로그 |
| 2 | MES | 공정/설비/제작번호 매핑 후 실적 기록 | 생산실적 레코드 |
| 3 | OPERATOR | 자동 수집 실패 시 수동 보정 입력 | 실적 보정 이력 |
| 4 | MES | 제작번호 기준 공정 진행 상태 갱신 | 공정 진행 상태 |
| 5 | MES | 대시보드 / 생산 현황판에 반영 | 집계 KPI |
| 6 | PROD_MANAGER | 일마감 확정 | 일마감 실적 레코드 |

### TG 수집 방식 상세

- **방식**: 이탈리아 TG기 출력단에서 접점(Dry Contact) 신호 추출
- **카운트 기준**: 1회 신호 = 1타격 완료
- **생산량 역산**: `타격 횟수 × 피치(mm) = 생산 길이(m)` → m² 환산
- **보정 허용**: 자동 수집 값과 수동 입력 값이 다를 경우 보정 이력 저장
- **신뢰도 기준**: 100% 정확도 불필요, 운영 기준 오차 ±3% 이내 허용

### 예외 및 확인 포인트

- 접점 신호 단절 시 수동 입력으로 전환하고 알림 발생
- 일마감 후 소급 보정은 사유 기록 필수
- TG 센서/바코드 보완 구조는 2차 검토 사항

| 단계 | 역할 |
| --- | --- |
| 부적합 발생 감지 | OPERATOR |

| 5 | QUALITY | 품질 분석 화면에서 추세 조회 | - |

| 유형 코드 | 부적합 유형 | 적용 공정 |
| --- | --- | --- |
| NC-01 | 치수 불량 | TG, 포밍 |

### 예외 및 확인 포인트

- 저울 연동 미구현 시 수기 입력 + 파일 업로드 대안 제공
- 고객사 성적서 자동 발행 범위는 1차 구축 후 별도 확정
- 부적합률 계산식: `부적합 중량 / 총 생산 중량 × 100 (%)`

## PROC-04 야적장 적재 및 출하 조회

### As-Is / To-Be

| 구분 | As-Is | To-Be |
| --- | --- | --- |
| 위치 관리 | 없음 (경험과 기억 의존) | 현장명 + 슬리퍼 번호 + 패킹번호 기준 등록 |
| 출하 탐색 | 수십 분 탐색 소요 | 조건 검색으로 즉시 위치 확인 |

### 적재 위치 키 체계

```
현장명 (예: 강남구청 현장)
  └── 슬리퍼 번호 (예: SL-0042)
        └── 패킹번호 (예: PK-2026-0001)
              └── 야적장 위치번호 (예: Y-A03-02)
```

### MES 처리 흐름

| 단계 | 행위자 | 동작 | 생성/변경 데이터 |

| 3 | MES | 적재 재고 상태 갱신 | 재고 상태 |
| 4 | SHIPPING | 패킹번호 또는 현장명으로 위치 조회 | - |
| 5 | SHIPPING | 출하 완료 후 상태 변경 | 출하 이력 레코드 |
| 6 | MES | 재고에서 차감 | 재고 상태 |

### 재고 상태 정의

| 상태 코드 | 상태명 | 설명 |
| --- | --- | --- |

| ST-03 | 출하 준비 | 출하 지시 수신 |

### 예외 및 확인 포인트

- Location Map은 2차 고도화 대상, 1차는 목록/카드 조회로 운영

- 야적장 세부 구획 작업 범위는 별도 확정 필요

## PROC-05 추적관리 및 원자재/로트 이력 조회

### As-Is / To-Be

| 구분 | As-Is | To-Be |
| --- | --- | --- |
| 원자재 관리 | FIFO 관리 없음 | 입고 시 라벨/로트 등록, 선입선출 기준 관리 |
| 불량 역추적 | 불가 | 제작번호 기준 원자재→출하 전 이력 연결 조회 |
| 현장별 추적 | 없음 | 현장명 기준 투입 이력 역추적 |

### 추적 키 연결 구조

```
원자재 로트번호 (LOT-######)
    ↓ [신선 공정 투입]
제작번호 (WORK-######) ← 전체 공정 추적의 중심 키
    ├── TG 공정 실적
    ├── 포밍 공정 실적
    ├── 부적합 실적
    ├── 라벨 출력 이력
    ├── 조립 완료 / 패킹번호
    └── 야적장 위치 / 출하 이력
```

### MES 처리 흐름

| 단계 | 행위자 | 동작 | 생성/변경 데이터 |
| --- | --- | --- | --- |
| 1 | OPERATOR | 원자재 입고 시 라벨 부착 및 로트 등록 | 원자재 로트 레코드 |
| 2 | MES | 공정 투입 시 제작번호와 로트 자동 연결 | 투입 이력 |
| 3 | MES | 공정별 실적 기록 시 제작번호에 누적 | 공정별 이력 |
| 4 | MES | 조립, 패킹, 출하까지 동일 제작번호로 이력 연결 | 통합 이력 |
| 5 | PROD_MANAGER | 제작번호 또는 로트 기준으로 전체 이력 조회 | - |
| 6 | PROD_MANAGER | 불량 발생 건의 원인 및 영향 범위 확인 | - |

### 예외 및 확인 포인트

- 수동 입력 공정의 이력 누락 방지 규칙 필요 (필수 입력 강제)
- 로트 기준 FIFO 위반 건은 시스템 경고 표시
- 현장명, 슬리퍼, 패킹번호, 위치번호의 키 체계 정합성 유지 필수

### MES 처리 흐름

| 단계 | 행위자 | 동작 | 생성/변경 데이터 |

| 3 | MES | 30분 초과 시 FACILITY / PROD_MANAGER 알림 발송 | 알림 이벤트 |
| 4 | FACILITY | 정지유형 입력 (고장/계획/기타) | 정지 이력 레코드 |
| 5 | MES | 정지유형 입력 확인 후 재가동 허용 | 가동 재개 타임스탬프 |
| 6 | MES | 가동/정지 이력 기반 MTBF 갱신 | MTBF 지표 |

### 정지유형 분류

| 유형 코드 | 유형명 | 설명 |
| --- | --- | --- |
| ST-F | 고장 정지 | 설비 고장으로 인한 비계획 정지 |
| ST-P | 계획 정지 | 청소, 점검, 코일 교체 등 계획 작업 |
| ST-B | 대기 정지 | 자재 부족, 작업 지시 없음 |
| ST-R | 휴게 정지 | 점심, 교대 등 휴게시간 |
| ST-O | 기타 정지 | 그 외 사유 |

### MTBF 산출 기준

```
MTBF = 총 가동 시간 / 고장(ST-F) 발생 횟수

총 가동 시간 = 실적 집계 시간 - ST-F / ST-P / ST-O 정지 시간
(ST-R 휴게 정지는 분모/분자 모두 제외)
```

### 예외 및 확인 포인트

- 휴게시간(ST-R) 제외 기준을 교대 스케줄과 연동하거나 수동 설정
- 안돈(물리적 신호등/버저)과 시스템 알림의 역할 분담: 안돈은 현장 즉각 시각/청각 알림, 시스템 알림은 사무실 담당자 앱/화면 알림
- 정지유형 입력을 시스템 강제 규칙으로 설정할지 현장 운영 기준 확정 필요
