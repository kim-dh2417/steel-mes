# 성지제강 공정 및 MES 도메인 분석서

## 문서 목적

본 문서는 성지제강 데크 3공장의 운영 문제와 공정 구조를 기준으로, 이번 MES 프로젝트가 무엇을 어떻게 해결하려는지 빠르게 설명하기 위한 발표/공유용 문서입니다.

## 1. MES 이해

### MES란 무엇인가

MES는 `Manufacturing Execution System`의 약자입니다.  
생산 현장에서 발생하는 데이터를 수집하고, 생산실적, 공정 진행, 설비 상태, 품질, 재고 흐름을 하나의 시스템에서 관리하는 제조 실행 시스템입니다.

### MES 핵심 모듈 이해

#### 개요

> MES는 크게 생산실적 관리, 공정 추적, 설비 관리, 재고 관리 4개 모듈로 볼 수 있습니다.  
> 각 모듈은 서로 데이터를 주고받으며 하나의 운영 흐름을 만듭니다.

#### 1. 생산실적 관리

> 오늘 무엇을 얼마나 생산했는지 기록하고 집계하는 모듈

- 공정별 생산 수량 자동 집계
- 제작번호별 진행 현황 조회
- 일/주/월 단위 생산실적 리포트
- 현장과 사무실에서 동일하게 보는 생산 현황판

#### 2. 공정 추적

> 제품이 신선 -> TG -> 포밍 -> 조립을 거치며 어디까지 진행되었는지 추적하는 모듈

- 제작번호별 공정 진행 현황 조회
- 불량 발생 시 자재, 작업지시, 수주정보 역추적
- 배치/로트 단위 이력 연결
- 현장명, 슬리퍼, 패킹번호, 라벨 기반 추적

#### 3. 설비 관리

> 설비가 얼마나 가동되고 멈췄는지, 왜 멈췄는지 관리하는 모듈

- 설비별 가동/정지 이력 기록
- MTBF 리포팅
  - 설비의 가동시간과 정지횟수를 기준으로 보는 운영 지표
- 30분 이상 정지 시 자동 알림
- 정지유형 입력 후 재가동 관리
- 안돈과 설비 상태 알림
  - 설비 이상이나 고장 발생 시 현장과 사무실에 즉시 알리는 경보 체계

#### 4. 재고 관리

> 원자재 입고부터 완제품 적재와 출하까지 재고 흐름을 관리하는 모듈

- 원자재 입고 라벨 및 로트 관리
- 선입선출 관리
- 야적장 위치번호 및 패킹번호 관리
- 적재 재고 및 출하 현황 조회

#### 모듈 간 연결 흐름

```text
원자재 입고(재고관리)
    ↓
작업지시 및 생산 준비
    ↓
공정별 생산(생산실적 관리 + 공정 추적)
    ↓
설비 모니터링(설비 관리)
    ↓
포장/적재/출하(재고 관리 + 공정 추적)
```

### 설비-시스템 연결 방식

#### 개요

> 공장 설비가 MES와 어떻게 데이터를 주고받는지를 결정하는 방식입니다.  
> 연결 방식에 따라 자동화 수준과 구현 난이도가 달라집니다.

#### 1. OPC

> PLC나 설비 데이터를 MES로 표준 방식으로 전달하는 연결 방식

- 설비나 PLC에서 생산수량, 가동상태 같은 데이터를 실제로 꺼내 MES가 읽을 수 있어야 합니다.

적용 포인트:
- 신선, 포밍, 조립 공정은 OPC 기반 자동집계 우선 검토 대상입니다.
- TG는 외산 설비라 직접 OPC 연동보다 대체 수집 방식을 우선 검토합니다.

#### 2. 바코드

> 제품이나 자재에 부착한 라벨을 작업자가 스캔해 MES에 기록하는 반자동 방식

- 작업자가 스캔하면 MES에 기록되는 반자동 방식입니다.
- 수기 입력보다 정확하고, 구현 난이도와 비용이 낮습니다.

적용 포인트:
- TG 보완 수집 방식으로 활용합니다.
- 원자재, 로트, 패킹번호, 야적장 추적에 활용합니다.

#### 3. 센서

> 설비 출구나 공정 지점에서 제품 통과 또는 동작 횟수를 감지해 카운트하는 반자동 방식

- 제품 통과나 동작 횟수를 감지해 자동 카운트하는 방식입니다.
- 저비용 자동화 수단이지만, 설비 내부 데이터만큼 정밀하지는 않습니다.

적용 포인트:
- TG 설비에서 OPC가 안 될 경우 접점 신호, 센서 카운트 방식이 핵심 대안입니다.
- 자동 집계와 수동 보정 구조를 같이 설계해야 합니다.

#### 4. 수동 입력

> OP가 직접 실적 입력하거나 보정 입력할 수 있는 방식

- 공정별 직접 입력 요구를 반영해야 합니다.
- 자동 수집이 어려운 구간이나 예외 상황에서 보정 입력이 필요합니다.

#### 방식별 비교

| 방식 | 자동화 수준 | 구현 난이도 | 비용 | 데크 3공장 적용 방향 |
| --- | --- | --- | --- | --- |
| OPC | 높음 | 높음 | 높음 | 신선/포밍/조립 우선 검토 |
| 바코드 | 중간 | 낮음 | 낮음 | TG 대체안, 원자재/로트/야적장 관리 |
| 센서 | 중간 | 낮음 | 낮음 | TG 보완책, 카운트 수집 |
| 수동 입력 | 낮음 | 낮음 | 낮음 | OP 직접 입력, 보정 입력 |

## 2. 성지제강 분석

### 성지제강의 회사와 현장 맥락

- 성지제강은 다수 공장을 운영하고 있으며, 이번 구축 대상은 데크 3공장입니다.
- 3공장은 주문생산과 분할납품 특성이 강한 공장입니다.
- 주문생산 구조라 추적 및 이력 관리가 중요합니다.
- 분할납품 구조라 1~2주 단위로 나눠서 보내는 물량의 위치와 출하 상태를 계속 확인해야 합니다.
- 따라서 이 공장은 생산량 집계보다 `추적`, `야적장 위치`, `출하 연결`, `라벨 식별`이 더 중요합니다.
- 현재는 엑셀과 이메일 중심 운영이라 생산, 추적, 품질, 재고, 설비 가시성이 부족합니다.
- 현재 방향은 `3공장 MES 우선 -> ERP 연동 -> 전사 확산`입니다.

### 성지제강의 공정 이해

#### 성지제강의 전체 공정 흐름

> 원자재(선재) -> 신선 -> TG -> 포밍 -> 조립 -> 포장 -> 야적장

#### 1. 신선

> 굵은 금속을 가늘게 뽑아내는 공정

- 자동화 가능성이 높아 실적 집계 우선 후보로 볼 수 있습니다.

MES 연계 포인트:
- 신선 공정 실적 자동 집계

#### 2. TG

> 여러 가닥의 선을 꼬아서 하나로 합치는 공정

- 중간 핵심 설비라 병목이 되기 쉬운 공정입니다.
- 외산(이탈리아산) 장비라 데이터 접근이 제한적입니다.
- 이 프로젝트에서 가장 중요한 수집 리스크가 집중되는 구간입니다.

MES 연계 포인트:
- 접점 방식 적용
- 필요 시 센서, 바코드 방식으로 보완
- 생산량 역산과 가동/정지 상태 수집
- 부적합 발생 시 즉시 등록

#### 3. 포밍

> 선재를 원하는 형상으로 성형하는 공정

- 제품의 최종 형상을 결정하는 핵심 가공 공정입니다.
- 제품 형상이 결정되는 구간이라 부적합 관리 중요도가 높습니다.
- 기존 부적합 측정 저울 연계 검토가 필요합니다.

MES 연계 포인트:
- 생산실적 자동 집계
- 부적합 등록 및 분석
- 저울 데이터 연동 검토

#### 4. 조립

> 각 공정 부품을 합쳐 최종 완제품을 만드는 마지막 공정

- 외주 운영으로 작업지시, 진행상태 확인, 알림, 패킹, 야적장, 출하까지 관리 포인트가 많은 공정입니다.
- 조립 완료 이후 패킹, 라벨링, 야적장 적재, 출하 준비가 이어집니다.
- 패킹번호와 야적장 위치까지 함께 연결돼야 합니다.

MES 연계 포인트:
- 패킹번호 기준 실적 처리
- 외주 작업지시 및 진행상태 공유
- 야적장 위치 입력 및 출하 연동

#### 성지제강의 공정별 자동화 목표 요약

| 공정 | 현재 | 목표 | 비고 |
| --- | --- | --- | --- |
| 신선 | 수동 입력 | OPC 자동집계 | 자동화 우선 후보 |
| TG | 수동 입력 | 접점 신호 기반 수집 + 보완 수단 검토 | 외산 설비 데이터 비공개 |
| 포밍 | 수동 입력 | OPC 자동집계 | 저울 연동 검토 |
| 조립 | 수동 입력 | 실적 집계 + 패킹/야적 연계 | 외주 공정 관리 포함 |

### 성지제강의 문제와 MES 적용 방향

| 회사의 문제 | 현장에서 필요한 대응 | MES 적용 방향 |
| --- | --- | --- |
| 전산 시스템이 없어 생산지시, 실적 집계, 보고가 엑셀과 이메일 중심으로 운영됩니다. | 공정별 생산실적 자동 집계와 제작번호별 공정 진행 조회가 필요합니다. | 설비 신호, 센서, 바코드 기반으로 공정 실적을 자동 수집하고 제작번호 기준 공정 진행 현황을 조회합니다. |
| 주문생산과 분할납품 구조라 추적이 어렵습니다. | 원자재부터 철선, TG, 조립, 출하까지 이력 추적이 필요합니다. | 제작번호, 현장명, 슬리퍼, 라벨, 패킹번호를 하나의 추적 키 체계로 연결합니다. |
| 야적장 위치 관리가 안 되어 출하 시 제품을 찾는 데 시간이 오래 걸립니다. | 현장명, 슬리퍼 번호, 적재/출하 현황 조회가 필요합니다. | 패킹번호와 위치정보를 연결하고 야적장 재고 위치와 출하 대상을 즉시 조회할 수 있게 합니다. |
| 수율과 부적합 집계가 수작업이라 정확도와 속도가 떨어집니다. | 부적합 등록, 부적합률 계산, 고객사 성적서 대응이 필요합니다. | 부적합 실적 등록과 중량 기반 품질 지표 계산 구조를 만들고 성적서 출력 기반 데이터를 정리합니다. |
| TG기와 주요 설비 상태를 사무실에서 즉시 확인하기 어렵습니다. | TG기 가동/정지 확인, 안돈, 30분 이상 정지 알림, MTBF가 필요합니다. | 설비 상태 모니터링, 정지 알림, 정지유형 입력, MTBF 리포트 기능을 구축합니다. |
| TG기는 제조사 데이터가 비공개라 직접 수집이 어렵습니다. | TG기 생산량과 가동상태를 간접 방식으로라도 집계해야 합니다. | TG기 출력단 접점 신호를 추출해 카운트 기반으로 생산량과 가동 상태를 집계합니다. |
| 분진 환경과 공간 제약으로 현장 PC 설치가 어렵습니다. | 사무실과 현장이 함께 볼 수 있는 운영 구조가 필요합니다. | 현장 단말 최소화, 사무실 중심 모니터링, 원격 접속 기반 화면 운영을 전제로 설계합니다. |
| 조립 공정이 외주 운영이라 현장 지시와 진행 확인이 끊깁니다. | 외주 조립 공정에 대한 지시 전달과 진행 모니터링이 필요합니다. | 작업지시, 알림, 처리 상태를 MES 화면에서 공유할 수 있는 외주 공정 관리 흐름을 포함합니다. |
| 현재는 3공장 우선이지만 이후 ERP 연동과 전사 확산까지 고려해야 합니다. | 1차는 3공장 성공, 이후 ERP 연동과 타 사업부 확산이 가능해야 합니다. | 3공장 MES를 우선 구축하되, 생산·품질·재고·출하·설비 데이터를 ERP 연계 가능한 구조로 관리합니다. |

## 3. 이번 프로젝트 적용 방향

### 이번 프로젝트에서 특히 중요한 포인트

- `TG기 수집 방식`이 선행 과제입니다. TG기 데이터가 안 잡히면 생산실적 자동 집계 신뢰도도 같이 흔들립니다.
- 이 프로젝트의 본질은 `생산 집계`가 아니라 `추적 가능한 운영 체계`입니다. 생산, 라벨, 야적장 위치, 출하, 추적관리가 하나의 흐름으로 연결되어야 합니다.
- 설비 알림은 단순 경보가 아니라 `운영 통제` 수단입니다. 30분 이상 정지 알림, 정지유형 입력, MTBF까지 같이 관리되어야 합니다.
- 1차 구축은 3공장 중심이지만 구조는 확장 가능해야 합니다. 이후 ERP 연동과 타 사업부 확산까지 고려해야 합니다.

### 이번 MES가 해결하려는 범위

#### 1차 우선 해결 범위

- 공정별 생산실적 자동 집계
- 제작번호별 공정 진행 조회
- 야적장 위치 관리 및 출하 연계
- 원자재/반제품/완제품 추적관리
- 부적합 등록 및 품질 실적 관리
- 설비 상태 모니터링, 안돈, 정지 알림, MTBF
- 공정별 라벨 출력

#### 후속 확장 관점

- ERP 연동
- 타 사업부 확산
- 고객사 성적서 자동화 고도화
- 외주 공정 관리 정교화
- 공정전환횟수, 인원별 실적 등 심화 지표

## 결론

- 이번 성지제강 MES 프로젝트는 단순한 생산 집계 시스템 구축이 아닙니다.
- 성지제강 데크 3공장의 `생산`, `추적`, `야적`, `품질`, `설비` 문제를 하나의 운영 체계로 연결하는 프로젝트입니다.
- 따라서 구현 우선순위는 `TG기 수집 기반 확보 -> 생산실적/추적/야적장 관리 -> 품질/설비 운영 체계 연결` 순으로 잡는 것이 타당합니다.
- 이 구조가 잡혀야 이후 ERP 연동과 사업부 확산도 의미 있게 이어질 수 있습니다.
