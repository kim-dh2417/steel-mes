## FEAT-01 생산실적 자동/반자동 집계

### 기능 개요

공정별 생산실적을 자동 또는 반자동으로 수집하고 기록합니다.

### 기능 목적

수기 입력 중심 실적 집계를 줄이고 실시간 현황 기반을 만듭니다.

### 주요 사용자

OPERATOR, PROD_MANAGER

### 입력 데이터

| 데이터 | 출처 | 유형 |
| --- | --- | --- |
| 접점 신호 (TG기) | 설비 출력단 | 자동 (접점 수집) |
| OPC 실적 데이터 (신선/포밍) | PLC OPC 서버 | 자동 |
| 수동 보정 수량 | OPERATOR 입력 | 수동 |
| 제작번호 | 작업지시 연동 | 참조 |
| 설비 ID | 설비 마스터 | 참조 |

### 출력 데이터

| 데이터 | 대상 화면 / 기능 |
| --- | --- |
| 공정별 생산실적 레코드 | 생산실적 조회, 생산 대시보드 |
| 제작번호별 공정 진행 상태 | 공정 진행 조회 |
| 실시간 KPI 집계 | 생산 현황판 |
| 보정 이력 | 감사 로그 |

### 상세 동작

1. 설비 또는 입력 수단에서 생산 이벤트를 수신합니다.
2. 공정, 설비, 제작번호 기준으로 실적을 기록합니다.
3. 자동 수집 실패 시 수동 보정 입력을 허용합니다.
4. 일마감 후 확정 처리하고 집계 결과를 대시보드에 반영합니다.

### 수용 기준 (Acceptance Criteria)

- 접점 신호 기반 TG 생산실적이 1분 이내에 시스템에 반영된다.
- 수동 보정 시 보정 사유와 이전 값을 이력으로 보존한다.
- 일마감 전/후 구분하여 집계 결과를 조회할 수 있다.

### 예외 및 제약

- TG는 접점 신호 기반 수집을 우선 적용하고, 센서/바코드 보완 구조는 2차 검토
- OPC 연결 단절 시 자동 알림 발생 및 수동 입력 전환

## FEAT-02 제작번호별 공정 진행 조회

### 기능 개요

제작번호 기준으로 현재 공정 상태를 조회합니다.

### 기능 목적

생산 중 제품이 어디까지 진행됐는지 즉시 확인할 수 있게 합니다.

### 주요 사용자

OPERATOR, PROD_MANAGER, VIEWER

### 입력 데이터

| 데이터 | 출처 | 유형 |
| --- | --- | --- |
| 제작번호 (검색어) | 사용자 입력 | 검색 조건 |
| 현장명 (검색어) | 사용자 입력 | 검색 조건 |
| 공정 상태 필터 | 사용자 선택 | 필터 조건 |

### 출력 데이터

| 데이터 | 설명 |
| --- | --- |
| 공정 진행 상태 목록 | 신선/TG/포밍/조립 각 공정 상태 |
| 마지막 생산품 정보 | 최종 실적 시간, 수량 |
| 공정 상태 배지 | 대기 / 진행중 / 완료 / 보정 필요 |

### 공정 상태 정의

| 상태 코드 | 상태명 | 설명 |
| --- | --- | --- |
| PS-01 | 대기 | 작업지시 생성, 미착수 |
| PS-02 | 진행중 | 해당 공정 실적 수집 중 |
| PS-03 | 완료 | 공정 완료 확정 |
| PS-04 | 보정 필요 | 자동 수집 실패 또는 이상값 |

### 수용 기준

- 제작번호 검색 결과가 3초 이내에 반환된다.
- 각 공정별 상태와 최종 실적 시간이 함께 표시된다.

### 예외 및 제약

- 자동집계가 누락된 경우 `보정 필요` 상태 자동 표시

## FEAT-03 부적합 등록

### 기능 개요

TG/포밍/조립 공정의 부적합을 현장에서 즉시 등록합니다.

### 기능 목적

현장에서 발생한 품질 문제를 즉시 데이터화하여 집계와 추적을 가능하게 합니다.

### 주요 사용자

OPERATOR, QUALITY

### 입력 데이터

| 필드 | 유형 | 필수 | 설명 |
| --- | --- | --- | --- |
| 제작번호 | 텍스트 | Y | 관련 작업지시 번호 |
| 공정 | 선택 | Y | 신선/TG/포밍/조립 |
| 부적합 유형 | 선택 | Y | NC-01~NC-05 코드 참조 |
| 수량 | 숫자 | N | 수량 기준 (개) |
| 중량 | 숫자 | N | 중량 기준 (kg) — 수량 또는 중량 중 하나 필수 |
| 메모 | 텍스트 | N | 상세 설명 |

### 출력 데이터

| 데이터 | 대상 |
| --- | --- |
| 부적합 실적 레코드 | 부적합 분석 화면 |
| 중량 기반 부적합률 | 품질 대시보드 KPI |
| QUALITY 알림 이벤트 | 알림 화면 |

### 수용 기준

- 공정, 제작번호, 부적합 유형, 수량/중량 중 하나 입력 시 저장 가능하다.
- 저장 즉시 QUALITY 역할 담당자에게 알림이 발송된다.
- 중량 기반 부적합률이 자동 계산되어 표시된다.

### 예외 및 제약

- 저울 연동이 없으면 수기 입력 또는 파일 업로드 대안 제공
- 동일 제작번호에 복수 부적합 등록 가능

## FEAT-04 부적합 분석 조회

### 기능 개요

부적합 실적을 기간 및 공정별로 집계하여 분석합니다.

### 기능 목적

품질 문제 추세를 빠르게 파악하고 성적서 대응 기반을 만듭니다.

### 주요 사용자

QUALITY, PROD_MANAGER, VIEWER

### 입력 데이터

| 필드 | 유형 | 설명 |
| --- | --- | --- |
| 기간 | 날짜 범위 | 일/주/월 단위 조회 |
| 공정 | 선택 | 전체 또는 특정 공정 |
| 부적합 유형 | 선택 | 전체 또는 특정 유형 |

### 출력 데이터

| 데이터 | 설명 |
| --- | --- |
| 기간별 부적합 건수 | 일/주/월 집계 |
| 공정별 부적합률 | 중량 기반 % |
| 부적합 유형별 분포 | 차트 |
| 상세 건 목록 | 클릭 시 추적 조회 연계 |

### 수용 기준

- 일/주/월 단위 분석이 단일 화면에서 전환 가능하다.
- 특정 부적합 건 클릭 시 추적 조회 화면으로 이동한다.

### 예외 및 제약

- 품질팀 알림 연계 범위와 고객사 성적서 자동 발행 범위는 별도 확정 필요

## FEAT-05 라벨 출력 및 KS 라벨 발행

### 기능 개요

작업지시 기준 공정별 라벨과 신선 공정 KS 라벨을 출력합니다.

### 기능 목적

공정별 출력 기준을 통일하고 추적 키를 현장에 물리적으로 연결합니다.

### 주요 사용자

OPERATOR

### 입력 데이터

| 필드 | 유형 | 필수 | 설명 |
| --- | --- | --- | --- |
| 공정 | 선택 | Y | 신선/TG/포밍/조립 |
| 설비 ID | 선택 | Y | 공정 선택 후 해당 설비 목록 |
| 제작번호 | 텍스트/선택 | Y | 작업지시와 연결 |
| 라벨 종류 | 선택 | Y | 일반 / KS 라벨 |
| 출력 수량 | 숫자 | Y | 출력 매수 |

### 출력 데이터

| 데이터 | 설명 |
| --- | --- |
| 라벨 인쇄 데이터 | 라벨프린터로 전송 |
| 출력 이력 레코드 | 출력 시간, 담당자, 수량 |

### 라벨 항목 정의

**공정 공통 라벨 항목**: 현장명, 제작번호, 공정, 설비, 생산일, 작업자

**KS 라벨 추가 항목 (신선 전용)**: KS 규격 번호, 제품 종류, 굵기, 제조업체명, 로트번호

### 수용 기준

- 라벨 미리보기 후 출력이 가능하다.
- 필수 항목 누락 시 출력이 차단된다.
- 출력 이력이 자동 저장된다.

### 예외 및 제약

- 프린터 연결 실패 시 에러 메시지 및 재출력 절차
- KS 라벨 필수 항목은 현장 확인 후 최종 확정

## FEAT-06 생산 현황판 조회

### 기능 개요

현장과 사무실에서 같은 기준으로 생산 현황을 실시간 조회합니다.

### 기능 목적

생산량 공유 기준을 하나로 통일하고 현장-사무실 정보 격차를 줄입니다.

### 주요 사용자

OPERATOR, PROD_MANAGER, FACILITY, VIEWER (전 역할)

### 출력 데이터 (조회 전용)

| 데이터 | 설명 | 갱신 주기 |
| --- | --- | --- |
| 공정별 금일 생산량 | 신선/TG/포밍/조립 | 실시간 (1분) |
| TG 설비별 가동 상태 | 7대 각각 | 실시간 |
| 조립 라인별 생산량 | 5개 라인 | 실시간 |
| 금일 부적합 건수 | 합계 | 실시간 |

### 수용 기준

- 현황판은 로그인 없이 전용 화면 URL로 접근 가능하다 (사이니지 모드 지원).
- 데이터 갱신 주기 1분 이내이다.

## FEAT-07 야적장 위치 등록 및 조회

### 기능 개요

조립 완료 제품의 야적장 위치를 등록하고 조회합니다.

### 기능 목적

출하 시 제품 탐색 시간을 줄이고 재고 위치 관리 체계를 만듭니다.

### 주요 사용자

OPERATOR, SHIPPING, PROD_MANAGER

### 입력 데이터 (등록)

| 필드 | 유형 | 필수 | 설명 |
| --- | --- | --- | --- |
| 패킹번호 | 텍스트 | Y | 조립 완료 시 자동 생성 또는 수동 입력 |
| 현장명 | 선택 | Y | 수주 현장 |
| 슬리퍼 번호 | 텍스트 | Y | 패킹 단위 |
| 야적장 위치번호 | 텍스트 | Y | 예: Y-A03-02 |
| 적재 일시 | 날짜/시간 | Y | 자동 입력 또는 수동 |

### 출력 데이터 (조회)

| 데이터 | 설명 |
| --- | --- |
| 위치 정보 | 현장명 + 슬리퍼 + 패킹번호 + 위치번호 |
| 재고 상태 | ST-01~ST-04 상태 코드 |
| 적재 이력 | 등록 일시, 담당자 |

### 수용 기준

- 패킹번호 또는 현장명으로 위치를 3초 이내에 조회할 수 있다.
- 위치 미등록 건은 `적재 대기` 상태로 강조 표시된다.

### 예외 및 제약

- Location Map은 2차 고도화 대상, 1차는 목록/카드 조회

## FEAT-08 출하 현황 조회

### 기능 개요

적재 재고와 출하 진행 상태를 조회합니다.

### 기능 목적

출하 준비와 출하 현황을 한 번에 파악하여 출하 준비 지연을 방지합니다.

## FEAT-13 MTBF 조회

### 기능 개요

설비별 MTBF(평균 고장 간격)를 조회합니다.

### 기능 목적

설비 운영성과와 반복 고장 패턴을 지표로 확인합니다.

### 주요 사용자

FACILITY, PROD_MANAGER

### 출력 데이터

| 데이터 | 설명 |
| --- | --- |
| 설비별 MTBF | 시간 단위 |
| 기간별 MTBF 추이 | 월별 비교 |
| 고장 발생 빈도 | 설비별 |
| 정지유형별 분포 | 비율 |

### 구현 단계

2차 고도화

### MTBF 계산 기준

```
MTBF = 총 가동 시간 / 고장(ST-F) 발생 횟수
가동 시간 = 전체 시간 - ST-F - ST-P - ST-O 정지 시간
(ST-R 휴게 정지 제외)
```

### 수용 기준

- 설비별, 기간별 MTBF 조회가 가능하다.
- 고장 발생 빈도가 높은 설비가 상위 정렬된다.
